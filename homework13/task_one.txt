ID: DZ – 13T1
Требование: возможность редактирования чужого репозитория.
Модуль: Fork GitHub.
Имя: Fork репозитория.
Предусловие: 
1.	Открыть Google Chrome. 
2.	В строку поиска ввести https://github.com/. 
3.	Войти в свой репозиторий: нажать на Sign in (в правом верхнем углу), ввести логин и пароль.
Шаги:
1.	Открыть необходимый репозиторий пользователя.
2.	Сделать ответвление: нажать на кнопку Fork в верхнем правом углу.
3.	Создать копию репозитория в своем аккаунте: нажать на кнопку Create fork. 
Ожидаемый результат: копия репозитория пользователя в нашем аккаунте.
Приоритет: Medium.
Постусловие: редактировать чужой код, создать Pull Request, соединить наш Pull Request с изначальным репозиторием.


ID: DZ – 13T2 
Требование: возможность синхронизироваться с удаленной веткой.
Модуль: Pull Request GitHub.
Имя: закрытие Pull Request из ветки test в ветку master.
Предусловие: 
1.	Открыть Google Chrome. 
2.	В строку поиска ввести https://github.com/.
3.	Войти в свой репозиторий: нажать на Sign in (в правом верхнем углу), ввести логин и пароль. 
4.	Создать новую ветку test. 
5.	В ветке создать python файл.
6.	В файле создать новую задачу.
7.	Добавить файл в индекс. 
8.	Создать коммит на новую задачу.
9.	Отравить закоммиченные изменения в удаленный репозиторий.
Шаги:
1.	В нашем репозитории выбрать вкладку Pull Request (запрос на объединение двух веток).
2.	Открыть новый Pull Request: нажать кнопку New pull reguest.
3.	Выбрать из какой ветки происходит перенос: нажать кнопку compare: master и выбрать ветку test. В результате получается из ветки compare: test в base: master.
4.	Нажать кнопку Create pull reguest два раза.
5.	Перейти в Pull reguests и выбрать вкладку Conversation. Перейти вниз вкладки Conversation. Найти кнопку Merge pull reguest. Для закрытия Pull reguest выбрать один из 3 вариантов - Rebase and Merge.
6.	Нажать кнопку confirm rebase and merge.
Ожидаемый результат: Pull Request закрыт.
Приоритет: Medium.
Постусловие: вернуться в локальный репозиторий (на локальную ветку master), синхронизироваться с локальной веткой master, удалить ветку test


ID: DZ – 13T3.
Требование: возможность изменения биографии пользователя.
Модуль: Profile GitHub.
Имя: Изменение Bio в профиле пользователя.
Предусловие: 
1.	Открыть Google Chrome. 
2.	В строке поиска ввести https://github.com/.
3.	Войти в свой репозиторий: нажать на Sign in (в правом верхнем углу), ввести логин и пароль.
Шаги:
1.	Открыть профиль пользователя: нажать на кнопку в верхнем правом углу.
2.	В списке выбрать имя профиля (Signed in as “имя профиля”).
3.	Открыть вкладку Overview.
4.	Под avatar пользователя нажать кнопку Edit profile.
5.	Изменить биографию в окне Bio.
Ожидаемый результат: появление корректной биографии.
Приоритет: Medium.
Постусловие: сохранить изменения.
